# Документация

## Описание ZeldaGeoGuessr

ZeldaGeoGuessr — это игра, основанная на карте из игры The legend of Zelda: BOTW. Игроку показывают изображение случайного места в мире, и он должен угадать, где именно находится это место, поставив метку на карте.

## Правила игры

### Раунд

1. Вам показывают скриншот локации из игры.
2. Вы ставите метку на карте, указывая предполагаемое местоположение.
3. Начисляются очки — чем ближе метка к реальной точке, тем больше очков.

**Максимум за раунд — 5 000 очков.**

## Подсчёт очков

Очки за раунд рассчитываются по формуле:

$$S = 5000 \times e^{-d \,/\, D}$$

где **d** — расстояние между вашей меткой и реальной точкой, D - калибровочный коэфициент (требует настройки).

## Игра

Игра состоит из **5 раундов**. Очки за каждый раунд суммируются.

**Максимум за игру — 25 000 очков.**

## Структура проекта

На данный момент в папке `locs` хранятся картинки с локациями для игры которые нужно угадать и json-файл с информацией о локации. (Подробнее в описании locations.json). В папке `map` хранится картинка с полной картой это большое изображение размером 108 МБ рамером 18000x15000 пикселй. В папке `map_tiles` хранится этоже изображение но разбитое на тайлы 512x512 px. `doc.md` этот файл.

## locations.json

Данный файл содержит информацию о локации.

Примерное содержание файла:

```json
{
  "series": [
    {
      "id": 1,
      "name": "Игра 1: Достопримечательности",
      "rounds": [
        {
          "image": "location_s01_i01.png",
          "x": 4500,
          "y": 3800
        },
        
        ...

        {
          "image": "location_s01_i05.png",
          "x": 14200,
          "y": 4100
        }
      ]
    },
    {
      "id": 2,
      "name": "Игра 2: Не так просто",
      "rounds": [ ... ]
    },
    {
      "id": 3,
      "name": "Серия 3: Сложные локации",
      "rounds": [ ... ]
    }
  ]
}
```

## Карта

- Размер: 18 000 × 15 000 px
- Тайлы: 512 × 512 px
- Сетка тайлов: 36 столбцов × 30 строк (нумерация с 000)
- Формат имени: tile_{row}_{col}.jpg

## Система координат

- Начало координат: левый верхний угол карты
- Ось X: слева направо (0 → 18000)
- Ось Y: сверху вниз (0 → 15000)
- Единица: пиксель

Это работает отлично. Хотелось бы только поправить что бы изображение открывалось с выравниванием по центру вьюпорта и сбрасывалось тоже по центру вьюпорта. И более корректно приближалось в центр вьюпорта или к курсору.
